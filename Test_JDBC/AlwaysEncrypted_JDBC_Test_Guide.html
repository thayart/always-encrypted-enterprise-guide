ทดสอบ Microsoft JDBC Driver กับ SQL Server Always Encrypted

วัตถุประสงค์: พิสูจน์ว่า Microsoft JDBC Driver สามารถ Encrypt Parameter ได้หรือไม่ โดยไม่เกี่ยวกับ Kafka Connect

1. เตรียมไฟล์

  • JDK 11+
  • mssql-jdbc-8.4.1.jre11.jar

2. สร้างไฟล์ TestAE.java


import java.sql.*;

public class TestAE {

    public static void main(String[] args) {

        try {

            Class.forName("com.microsoft.sqlserver.jdbc.SQLServerDriver");

            String url =
                "jdbc:sqlserver://YOURSERVER:1433;" +
                "databaseName=CoreClaim;" +
                "encrypt=true;" +
                "trustServerCertificate=true;" +
                "columnEncryptionSetting=Enabled;";

            String user = "YOURUSER";
            String password = "YOURPASSWORD";

            System.out.println(url);

            Connection conn = DriverManager.getConnection(url, user, password);

            PreparedStatement ps = conn.prepareStatement(
                "INSERT INTO ext.Person_Encrypt " +
                "(PersonId, TitleId, IdentityCard) " +
                "VALUES (?, ?, ?)");

            ps.setInt(1, 999999);
            ps.setInt(2, 1);
            ps.setString(3, "1234567890123");

            int rows = ps.executeUpdate();

            System.out.println("Rows inserted = " + rows);

            PreparedStatement ps2 = conn.prepareStatement(
                "SELECT IdentityCard FROM ext.Person_Encrypt WHERE PersonId=?");

            ps2.setInt(1, 999999);

            ResultSet rs = ps2.executeQuery();

            while(rs.next()){
                System.out.println("IdentityCard = " + rs.getString(1));
            }

            rs.close();
            ps2.close();
            ps.close();
            conn.close();

            System.out.println("SUCCESS");

        } catch(Exception e){
            e.printStackTrace();
        }
    }
}

3. Compile

javac -cp ".;mssql-jdbc-8.4.1.jre8.jar" TestAE.java

4. Run

java -cp ".;mssql-jdbc-8.4.1.jre8.jar" TestAE

การแปลผล

ผลลัพธ์ความหมาย
SUCCESS และอ่าน IdentityCard ได้Always Encrypted ทำงานถูกต้อง ปัญหาอยู่ที่ Kafka Connect
Operand type clashDriver ไม่ได้ Encrypt Parameter ให้ ตรวจ URL และ CMK
Certificate / CEK Errorตรวจ Certificate Store, Private Key และสิทธิ์

Checklist

  • ใช้ Microsoft JDBC Driver
  • URL มี columnEncryptionSetting=Enabled
  • URL มี encrypt=true
  • Certificate มี Private Key
  • CMK อยู่ใน Windows Certificate Store และ Process มีสิทธิ์อ่าน
